Fix persistent "Choose File" button not opening file picker on:
PDF Split, PDF Compress, PDF Rotate, PDF Watermark, PDF to Word,
MP4 to MP3, Audio Converter, Video Compress, GIF Maker, Video Trim, Video Merger.

1) Standardize markup near the upload box:
   - Ensure each page has:
     <button class="ff-choose-btn" type="button">Choose a File</button>
     <input class="ff-file-input" type="file" hidden />
     (Use multiple/accept attrs per tool as before.)

2) CSS guardrails (shared stylesheet):
   .ff-upload-area { position: relative; }
   .ff-choose-btn { position: relative; z-index: 5; }
   .ff-file-input {
     position: absolute; inset: 0; opacity: 0;
     z-index: 6; cursor: pointer; pointer-events: auto;
   }
   /* Remove blockers */
   .upload-overlay, .decorative-gradient {
     pointer-events: none !important;
   }

3) JS binding (run on DOMContentLoaded AND after SPA page swaps):
   - For each tool page container (data-page-id present), do:
     const root = document.querySelector('[data-page-id]');
     root.querySelectorAll('.ff-choose-btn').forEach((btn) => {
       const input = btn.nextElementSibling?.matches('.ff-file-input')
         ? btn.nextElementSibling
         : root.querySelector('.ff-file-input');

       // Safety: rebind every time; no one-time flags.
       btn.onclick = (e) => {
         e.preventDefault(); e.stopPropagation();
         // Synchronous user gesture — no setTimeout/async before click
         input?.click();
       };

       input?.addEventListener('change', (e) => {
         if (!input.files || input.files.length === 0) return;
         // call the existing upload handler for this page
         window.ffUpload?.(input.files);
       });
     });

4) Kill double-open bug:
   - Remove ALL other listeners that call input.click() for the same button.
   - Search & remove legacy delegates like:
     document.addEventListener('click', e => { if(e.target.matches('.choose')) input.click() })
   - Ensure there is exactly ONE click → ONE input.click() path.

5) Remove blockers:
   - Inspect the upload section and delete/adjust any element with:
     position:absolute over the button, a higher z-index than 6, or pointer-events:auto.
   - Specifically check “Advanced Options” container and any gradient/halo wrappers.

6) Safari/iOS constraints:
   - Do NOT trigger input.click() inside a promise/setTimeout/await — must be in the same user gesture.
   - Make sure the <input type="file"> is NOT disabled and NOT display:none (hidden attribute is fine).

7) Test matrix per page:
   - Click “Choose a File” → picker opens exactly once; cancel → no reopen; choose → upload starts.
   - Drag & drop continues to work.
   - Mobile Safari and Chrome desktop verified.
